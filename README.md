# CIAF Watermarks

Forensic provenance and watermark verification for AI-generated artifacts.

CIAF Watermarks is a forensic provenance package for AI-generated artifacts. It combines watermarking, dual-state hashing, fragment-based verification, and evidence records to support authenticity analysis across text, images, PDFs, and emerging multimedia workflows.

**Watermarks can be removed. Evidence is harder to erase.**

## What It Is

CIAF Watermarks is a Python package for creating forensic evidence records for AI-generated content. It applies watermarks while preserving cryptographic proof of both the original artifact and the watermarked version.

The system addresses a core problem with traditional watermarking: watermarks are easy to remove. This package captures forensic fragments from the original content, enabling verification even when the watermark itself has been stripped or the content has been modified.

---

## Why It Matters

Ordinary watermarking asks whether a mark is present. CIAF Watermarks is designed to answer stronger questions:

- What was originally generated?
- What was distributed?
- Was attribution removed?
- Does the remaining artifact still match forensic evidence of the original?

This matters for:

- **Provenance tracking** - Verify origin of AI-generated content
- **Tampering detection** - Identify modifications after distribution
- **Watermark removal detection** - Prove attribution was stripped
- **Mix-and-match prevention** - Detect composite documents from multiple sources
- **Compliance** - Provide cryptographic audit trails for regulatory requirements

**Artifact Types**
- Text documents
- Images (PNG, JPEG, WebP)
- PDFs
- Video (MP4, AVI, MOV) - metadata and visual watermarks
- Audio (WAV, MP3, FLAC) - metadata and spectral analysis
- Binary files

**Watermark Modes**
- Metadata embedding
- Visible watermarks
- QR codes
- Steganographic (invisible)
- Hybrid (metadata + visible)

**Verification Methods**
- Exact hash matching (fastest, highest confidence)
- Forensic fragment matching (resilient to partial edits)
- Perceptual fingerprinting (resilient to format conversion)
- Hierarchical cascade (automatic fallback)

**Advanced Features**
- Dual-state hashing (pre-watermark + post-watermark)
- Fragment-based DNA sampling (3-6 strategic samples per artifact)
- GPU acceleration for batch processing
- Async processing for high-throughput workflows
- Vault integration for persistent storage
- Cryptographic signature envelopes

## Quick Start

### Unified Interface (Recommended)

Automatically detect artifact type and apply appropriate watermarking:

```python
from ciaf.watermarks import watermark_ai_output

# Watermark any artifact type
evidence, watermarked = watermark_ai_output(
    artifact=content,  # str, bytes, or Path
    model_id="gpt-4",
    model_version="2026-03",
    actor_id="user:analyst-17",
    prompt="Generate quarterly summary...",
    verification_base_url="https://vault.example.com"
)

# Save watermarked output
with open("output_watermarked.dat", "wb") as f:
    f.write(watermarked)

# Store evidence for later verification
vault.store_evidence(evidence)
```

### Text Example

```python
from ciaf.watermarks import build_text_artifact_evidence, verify_text_artifact

# Create watermarked text
evidence, watermarked_text = build_text_artifact_evidence(
    raw_text="AI-generated content...",
    model_id="gpt-4",
    verification_base_url="https://vault.example.com",
    enable_forensic_fragments=True
)

# Later: verify suspect text
result = verify_text_artifact(suspect_text, evidence)
print(f"Authentic: {result.is_authentic()}")
print(f"Confidence: {result.confidence:.1%}")
print(f"Fragments matched: {result.fragments_matched}/3")
```

## Verification Model

The package uses a three-tier verification strategy:

### Tier 1: Exact Hash Match

Fastest and highest confidence. If the suspect artifact's hash matches the stored post-watermark hash, verification succeeds immediately.

```python
if sha256(suspect_artifact) == evidence.state2_hash:
    confidence = 1.0  # Exact match
```

### Tier 2: Forensic Fragment Matching

Compares forensic DNA fragments extracted from strategic locations. If 2 out of 3 fragments match, confidence is 99%+.

```python
matched = count_matching_fragments(suspect, evidence.fragments)
if matched >= 2:
    confidence = 0.95+  # High confidence
```

### Tier 3: Perceptual Fingerprint

Resilient to format conversion, compression, and minor edits. Computes perceptual similarity.

```python
similarity = perceptual_similarity(suspect, evidence.fingerprint)
if similarity >= 0.90:
    confidence = 0.90  # Format-resilient match
```

**Key Insight**: Watermark removal is detectable because the hash will match State 1 (pre-watermark) instead of State 2 (post-watermark), providing cryptographic proof of tampering.

### Watermark Removal Detection

```python
# Original watermarked artifact
_, watermarked = watermark_ai_output(content, model_id="gpt-4", ...)

# User strips watermark
suspect = remove_watermark(watermarked)  # Malicious edit

# Verification detects tampering
result = verify_artifact(suspect, evidence)
# Result: is_authentic=False, reason="Hash matches pre-watermark state"
```

### Format Conversion Resilience

```python
# Watermark PNG image
evidence, watermarked_png = watermark_ai_output(png_bytes, ...)

# User converts to JPEG
jpeg_bytes = convert_to_jpeg(watermarked_png)

# Perceptual verification still succeeds
result = verify_artifact(jpeg_bytes, evidence)
# Result: is_authentic=True, confidence=0.92 (perceptual match)
```

## Security

All evidence records use cryptographic signature envelopes:

```python
{
    "envelope_version": "1.0",
    "payload": {
        "artifact_id": "art:abc123",
        "state1_hash": "sha256:...",  # Pre-watermark
        "state2_hash": "sha256:...",  # Post-watermark
        "forensic_fragments": [...],
        "timestamp": "2026-04-06T10:30:00Z"
    },
    "signature": {
        "value": "base64:...",
        "metadata": {
            "signature_algorithm": "Ed25519",
            "key_id": "aws-kms:alias/ciaf-prod",
            "canonicalization_version": "RFC8785-like/1.0"
        }
    }
}
```

**Features**
- Ed25519 elliptic curve signatures
- RFC 8785-compliant canonical JSON
- Key backend support: Local, AWS KMS, Azure Key Vault, HSM
- Complete audit trail
